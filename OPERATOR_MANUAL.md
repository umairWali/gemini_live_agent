# Personal AI Operator - User Manual & Architecture Overview

## Overview
This document describes the operation, monitoring, and architecture of your new autonomous AI partner. The system is designed as a persistent sidecar service that runs alongside your work, capable of safe, verified, and self-healing operations.

## Quick Start
The operator runs as a background service.

**Start the Service:**
```bash
cd sidecar
npm start
```
*Port 3000: IPC API (Commands)*
*Port 3001: Event Stream (Watchers)*

**Check Health:**
```bash
curl http://localhost:3000/health
```

## Core Capabilities (The "Brain")

### 1. Safety First (Judgment Engine)
Every action is risk-assessed before execution.
- **Low Risk:** (Read files, Check status) -> Auto-approved.
- **Medium/High Risk:** (Write files, Install packages, Delete) -> **Requires Confirmation** or explicit bypass flag.
- **Audit Log:** All decisions are recorded in `data/state.json`.

### 2. Self-Healing (Recovery Engine)
If a command fails (e.g., missing file, network error), the system will:
1. Classify the error type.
2. Attempt a known fix strategy (e.g., `npm install`).
3. Retry the operation automatically.

### 3. Stability Intelligence (Health Monitor)
If too many errors occur quickly, the system enters **SAFE MODE**.
- Autonomous actions are paused.
- Manual intervention is required to reset.
- protects your environment from runaway loops.

### 4. Reality Verification (Verifier Engine)
The system "trusts but verifies."
- **Pre-Check:** Ensures files exist before reading/deleting.
- **Post-Check:** Confirms files were actually written/deleted after success report.
- Detects **Drift** if the filesystem disagrees with the internal state.

### 5. Timing Intelligence (Priority Engine)
The system respects your workflow.
- Detects if you are in a **Meeting** or **Deep Work**.
- Defers low-priority tasks to a backlog.
- Budgets interruptions to minimize noise.

## File Structure
- `sidecar/server.ts`: Main entry point.
- `sidecar/modules/`: Intelligence engines (Judgment, Health, Priority, etc.).
- `data/state.json`: The "Memory" - audit logs, tasks, and system state.

## Troubleshooting
- **Logs:** Check the terminal where `npm start` is running.
- **Reset:** Stop the process (Ctrl+C) and restart. The system state persists in `data/state.json`.
